## Exact next patch I’d try  
  
### Core idea  
Keep the strong general post-evidence behavior, but add a **much narrower branch** only for:  
  
- post-evidence  
- partner-soft  
- non-narrow margin  
- **low-pressure** cases  
  
and then split:  
  
1. **cooperative-soft clearly dominant**    
   → allow `PROCEED`  
  
2. **opportunistic-soft still competitive**    
   → favor `PROBE`  
  
This is narrower than the earlier broad soft-partner routing because it only targets the **courteous / moderate / delayed-trust** style failures that seed 23 keeps surfacing.  
  
---  
  
## Why this is the best next patch  
Seed 23 keeps failing in:  
- `ambiguous_commit cooperative`  
- `false_friend cooperative`  
- `late_disambiguation cooperative`  
  
That suggests the missing behavior is not:  
- generic anti-overcommit  
- generic softness ambiguity  
  
It is more like:  
  
> **in low-pressure moderate cases, don’t trust softness unless cooperative belief clearly clears opportunistic belief.**  
  
So the patch should operate specifically there.  
  
---  
  
## Concrete train.py change shape  
  
In `_apply_decision_prior`, after:  
  
```python  
soft_partner_window = negotiation_window & evidence_released & partner_soft & (~margin_narrow)  
```  
  
add two new masks:  
  
```python  
delayed_trust_cooperative_resolution = (  
    soft_partner_window  
    & (~urgency_high)  
    & (~throughput_bias)  
    & (cooperative_belief > assertive_belief + 0.08)  
    & (cooperative_belief > opportunistic_belief + 0.18)  
).to(logits.dtype)  
  
delayed_trust_probe = (  
    soft_partner_window  
    & (~urgency_high)  
    & (~throughput_bias)  
    & (opportunistic_belief >= cooperative_belief - 0.02)  
).to(logits.dtype)  
```  
  
Then apply:  
  
```python  
logits[:, PROCEED] = logits[:, PROCEED] + 0.26 * delayed_trust_cooperative_resolution  
logits[:, PROBE] = logits[:, PROBE] - 0.16 * delayed_trust_cooperative_resolution  
logits[:, WAIT] = logits[:, WAIT] - 0.06 * delayed_trust_cooperative_resolution  
  
logits[:, ASSERT] = logits[:, ASSERT] - 0.12 * delayed_trust_probe  
logits[:, PROCEED] = logits[:, PROCEED] - 0.22 * delayed_trust_probe  
logits[:, WAIT] = logits[:, WAIT] + 0.04 * delayed_trust_probe  
logits[:, PROBE] = logits[:, PROBE] + 0.20 * delayed_trust_probe  
```  
  
And add diagnostics:  
  
```python  
diagnostics["delayed_trust_cooperative_resolution_mask"] = delayed_trust_cooperative_resolution  
diagnostics["delayed_trust_probe_mask"] = delayed_trust_probe  
```  
  
plus include those mask names in the existing mask export lists.
